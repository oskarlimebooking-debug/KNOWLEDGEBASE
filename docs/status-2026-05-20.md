## Feature details

### Phase T2 — Chapter progress in header

The chapter header is now a flex-column `.chapter-title-area`:
```
┌──────────────────────────────────────────────────────────────┐
│ ‹  Chapter 3: The Mind of a Strategist          🖊  ⬇  ✨  │
│    Ch 3 of 12 · 25% done                                    │
└──────────────────────────────────────────────────────────────┘
```

- `Ch X of Y` = current chapter's 1-based index in the book
- `Z% done` = chapters marked complete / total chapters × 100
- Updates automatically on each chapter navigation (openChapter() re-runs)
- `App._chapterCtx` now also carries `idx` and `total` for downstream use

### Phase N12 — URL article import

**UI flow:**
1. Click "🌐 From URL" in the library actions row
2. "Import from URL" modal appears with URL input
3. Enter URL (https:// prefix is added automatically if missing)
4. Click Import / press Enter
5. Progress banner: "Fetching article…" → "Saving…"
6. Book appears in library with the article title; one chapter = the full article text

**Serverless function** (`api/fetch-article/index.js`):
- GET with `?url=` query param
- Validates protocol (http/https only)
- Fetches with a browser-like User-Agent
- Extracts title from: `og:title` → `twitter:title` → `<title>` tag
- Extracts byline from: `author` meta → `article:author` → `twitter:creator`
- Extracts content from: `<article>` → `<main>` → div with class/id containing "content/article/post/entry" → `<body>`
- Strips all HTML tags + entities + excessive whitespace
- Falls back to full body if extracted content < 200 chars
- No npm dependencies (uses Node 18+ native fetch + regex)

**Book record** created with `kind: 'url'` and `sourceUrl` field for future reference.
