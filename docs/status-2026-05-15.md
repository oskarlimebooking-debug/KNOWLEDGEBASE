## Feature details

### N3 — Spaced Repetition (SM-2)

**IDB store: `review_cards`**
```
{
  id: 'srs_<hash>_<chapterId-tail>',
  chapterId, bookId,
  front: string,   // question (max 500 chars)
  back: string,    // answer  (max 500 chars)
  ease: number,    // default 2.5
  interval: number, // days until next review
  repetitions: number,
  nextReview: 'YYYY-MM-DD',
  lastReview: 'YYYY-MM-DD',
  createdAt: ISO
}
```

**SM-2 algorithm** (`sm2Update`):
- quality < 3 (Hard): reset repetitions=0, interval=1
- quality ≥ 3: interval grows (1→6→interval×ease), repetitions++
- ease adjusted: `ease + 0.1 - (5-q)*(0.08 + (5-q)*0.02)`, clamped ≥ 1.3

**Review session flow:**
1. `openReviewSession()` loads all cards with `nextReview ≤ today`
2. Full-screen overlay with card question → tap to flip → reveals answer
3. Three rating buttons: 😓 Hard (q=1) / 👍 Good (q=3) / ⚡ Easy (q=5)
4. Each rating calls `sm2Update()`, saves updated card to IDB, advances
5. After last card: summary screen — "N cards reviewed · X% felt easy"
6. Closing overlay calls `renderDailyAndStats()` to refresh badge + stats

**Enrol pathways:**
- **Flashcards tab**: "＋ Add to review deck" button below each card; turns green "✓ In review deck" when enrolled; state refreshed on Prev/Next
- **Quiz wrong answers**: after quiz summary, all wrong MC/TF questions auto-enqueued (idempotent); toast shows count of newly added cards

**Library integration:**
- `#btn-review` in header — hidden when due=0, shows amber badge with count when due>0
- Stats chip "🗂 N due for review" — clickable, also opens review session

## Known gaps / follow-ups

1. **Intra-day re-queue for Hard**: In Anki, "Hard" during initial learning resurfaces the card within the same session. Our implementation schedules it for tomorrow (interval=1). This is a deliberate simplification — revisit if users request same-day review.
2. **Review card management UI**: No way to browse, edit, or delete individual cards from the library. A "Review deck" modal listing all cards would complement the session.
3. **Cross-paragraph annotation offsets** (carried from N2).
4. **Heatmap dark-palette** hardcoded to dark-theme greens.
