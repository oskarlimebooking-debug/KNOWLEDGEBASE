### Implementation details

- `unmarkChapterComplete` extracts `bookId` from the `chapterId` string (`split('_ch_')[0]`), queries the `progress` IDB store by `bookId` index, then deletes all records where `p.chapterId === chapterId`. This removes completions across any date they were created.
- Button onclick now uses `isChapterComplete` (async re-read) rather than the captured `isDone` closure variable, so it's accurate even if the user marks/unmarks repeatedly without re-opening the chapter.
- Unmarking does **not** navigate — the user stays on the current chapter.
