## Feature details

### Badge lifecycle
1. **Chapter opens** → badge immediately clears to 0 (stale chapter's count gone), then async IDB query resolves and shows correct count
2. **User opens bookmark dropdown** → `_renderBookmarkMenu` refreshes the count (covers edge cases)
3. **User adds bookmark** → `addBookmark` fires non-blocking IDB re-count → badge increments
4. **User deletes bookmark** → `_renderBookmarkMenu` re-renders (already called on delete) → `_updateBmBadge` decrements; badge disappears at 0

### Visual spec
- Badge: `position: absolute; top: -3px; right: -3px; min-width: 16px; height: 16px; border-radius: 8px`
- Background: `var(--accent)` (amber in dark theme)
- Text: white, 10px, bold, centered
- `pointer-events: none` so badge doesn't interfere with button clicks
- Badge is **absent from DOM** when count = 0 (not hidden, actually removed)
