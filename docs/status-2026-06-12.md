### What was built today

**Phase N40 — Chapter search within book** (~25 lines added): The book view's "Chapters" tab now has a full-width filter input at the top. Typing in it instantly hides non-matching chapter rows (case-insensitive substring match on chapter title). If nothing matches, a "No chapters match" message appears. Clearing the input restores all rows. No IDB changes — pure DOM filtering.

**Bug encountered and fixed**: The initial implementation used `li.toggleAttribute('hidden', …)` but `.chapter-list li { display: flex }` in the CSS was overriding the browser's default `[hidden] { display: none }`. Fixed by switching to `li.style.display = show ? '' : 'none'`.

## Feature details

### N40 — Chapter search within book

- **Location**: Inside `<div id="book-tab-chapters">`, above the chapter list
- **Trigger**: `input` event (fires on every keystroke)
- **Match logic**: `title.toLowerCase().includes(query.toLowerCase().trim())`
- **No match state**: `.ch-search-empty` shown (`display: block`) when zero chapters are visible
- **Clear state**: When query is empty (`!q`), all chapters are shown
- **IDB changes**: None — pure DOM toggling
- **Bug fixed**: `toggleAttribute('hidden')` overridden by `display: flex`; switched to `style.display`
