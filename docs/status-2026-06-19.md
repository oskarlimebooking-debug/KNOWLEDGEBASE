## Current phase: N47 ✅ COMPLETED TODAY

### What was built today

**Phase N47 — Session history log** (~80 lines added):
Every completed SRS review session is now persisted to a new IndexedDB store `review_sessions` and viewable as a collapsible history panel at the bottom of the Deck Manager.

- **New IDB store**: `review_sessions` (keyPath `id`, indexes on `date` and `ts`). DB version bumped 6 → 7.
- **Auto-persist on session end**: when `renderSummary()` fires after a review, a record `{ id, date, ts, hard, good, easy, total }` is written to `review_sessions` (only when `reviewed > 0`, so empty close-outs are not logged).
- **Deck Manager history panel**: a collapsible "Session History (N)" section appears at the bottom of the deck body whenever at least one session exists. Collapsed by default (caret ▶ rotates to ▼ on expand). Shows last 10 sessions sorted newest-first.
- **Row layout**: `date | total | H/G/E bar | 😓N 👍N ⚡N` — colour-coded bar matches the N44 session stats palette (red/amber/green).

### Implementation details

- Session ID is `Date.now().toString(36) + Math.random().toString(36).slice(2,6)` — collision-free without a uuid library.
- `dbPut('review_sessions', ...)` is fire-and-forget (`.catch(() => {})`) so a storage error never interrupts the summary UI.
- History section only appears in deck manager body when sessions exist — zero-state deck manager is unchanged.
- Toggle is purely DOM (no new state variable) — collapsed state resets to closed every time `renderDeckMgr` is called (i.e. every time deck manager opens).
- Bar segments use the same colour vars as N44: `var(--bad)` / `var(--accent)` / `#34d399`.
