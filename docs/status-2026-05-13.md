## Feature details

### N6 — Reading position memory

- Scroll listener on `window` fires while in chapter view; debounced to one
  IDB write per 2 s via `setSetting('reading_pos_<chapterId>', { offset, ts })`.
- On chapter open: reads saved position. If ≥ 120 px, shows the Resume banner
  (dark panel between mode-tabs and chapter body) with **Resume** + **✕** buttons.
- **Resume** button smoothly scrolls to the saved offset and hides the banner.
- **✕** dismisses without scrolling.
- `_clearScrollSave()` removes the listener when navigating away via `openBook()`.

### N8 — Streak heatmap

- 53 × 7 SVG grid spanning the last ~365 days (aligned to Sundays).
- Each cell colored by chapter count per day: 0 = `#1a2340` (near-invisible), 
  1 = dark green, 2–3 = medium green, 4–5 = bright green, 6+ = brightest green.
- Future dates render as transparent cells.
- Tooltip on each cell shows date + count.
- The `allProgress` rows fetched in `renderDailyAndStats` are passed directly to
  `renderHeatmap` to avoid a second IDB read.
- SVG is responsive (fills container width via `width:100%`).

### N11 — Library sort options

| Option | Sort logic |
|---|---|
| Recently added (default) | `b.addedAt` descending (ISO string compare) |
| Recently read | Max `progress.ts` per book, descending |
| A–Z | `title.localeCompare` |
| Completion % | Pre-computed `pctMap` descending |

- Sort choice persisted to IDB via `setSetting('librarySort', value)`.
- Dropdown synced to the saved value on every `renderLibrary()` call.
- Changing the dropdown instantly re-renders the library.
