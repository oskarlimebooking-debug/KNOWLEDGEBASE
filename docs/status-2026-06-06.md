## Feature details

### Calibration flow

1. User opens Settings → Reading section → clicks **Calibrate**
2. Modal opens showing the sample paragraph (dimmed) + **Start reading** button
3. User clicks **Start** → paragraph brightens, timer begins
4. User reads at natural pace, clicks **Done** → elapsed time measured
5. WPM = `wordCount / elapsedMinutes` (rounded)
6. Result pane shows the WPM prominently + "Average adult reader: 200–250 WPM" hint
7. **Save** → writes to IDB via `setSetting('readingWpm', wpm)`, updates the WPM input in Settings, closes modal, shows "Reading speed saved: X WPM ✓" toast
8. **Try again** → resets to idle pane
9. **Cancel** / backdrop click → closes without saving

### Chapter header integration

Already wired from status-2026-06-05: `openChapter` reads `readingWpm` from IDB every time it loads, so the "~X min" estimate in the chapter header updates immediately after saving a calibrated value.
