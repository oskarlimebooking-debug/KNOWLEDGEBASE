## Feature details

### Confirmation flow
1. User selects cards → clicks "Delete (N)" → `selBar` content swaps to confirmation row
2. Confirmation row: `"Delete N cards?"` · red **Confirm** · grey **Cancel**
3. **Confirm** → executes the bulk `dbDelete` loop → clears `_deckSelected` → exits select mode → shows toast → re-renders
4. **Cancel** → calls `renderDeckMgr(filter)` → original select bar restored, selection preserved, no IDB change

### UX details
- No modal/overlay — entirely inline within the existing `.deck-sel-bar` flex row
- Confirmation row uses the same CSS classes as the normal bar (`deck-del-sel` for Confirm, `secondary` for Cancel) — no new CSS needed
- After Cancel, selection state is preserved (all N cards still checked)
- After Confirm, select mode exits and the bar disappears exactly as before
