- `package.json` — minimal scripts.
- `README.md` — user-facing readme with run/deploy instructions.

App functionality shipped:

- IndexedDB layer (`ChapterWiseDB` v1) with the five Phase A stores: `books`,
  `chapters`, `progress`, `generated`, `settings`. Indexes per spec
  (`books.addedAt`, `chapters.bookId`, `progress.bookId`, `progress.date`,
  `generated.chapterId`, `generated.type`).
- Wrapper helpers: `dbPut`, `dbBulkPut`, `dbGet`, `dbGetAll`, `dbGetByIndex`,
  `dbDelete`, `dbClear`, `getSetting`, `setSetting`. Exposed on
  `window.App` for debugging.
- ID conventions per spec: `book_<ts>` and `<bookId>_ch_<index>`.
- UI shell with three views (`library`, `book`, `chapter`) and a modifier-class
  view switcher. Sticky header per view, mobile-first responsive layout.
- Add Book flow:
  - PDF.js text extraction with line reconstruction by transform Y, and
    first-page cover render to a 300-wide JPEG.
  - EPUB extraction via JSZip — parses container.xml → OPF → spine, strips
    HTML, picks a cover image when present.
  - Auto-derives title from first usable line (PDF) or `<dc:title>` (EPUB).
  - Word-count chapter splitter (default 2000 words; configurable in Settings).
- Library view: book grid (cover or emoji fallback), per-book progress bar,
  empty state.
- Daily card: picks the next incomplete chapter from the most recently opened
  book, falling back to newest book.
- Stats row: streak chip, chapters complete, book count.
- Book detail view: hero with cover, completion bar, chapter list with
  completed-state styling, delete-with-confirm.
- Chapter view: paragraph rendering, prev/next nav, mark-complete with
  auto-advance.
- Settings modal: reading WPM, default chapter words, **Export** all data to
  JSON, **Import** from JSON (round-trips cleanly, merges by ID).
- Streak counter: walks backwards from today (or yesterday if no read today)
  through `progress.date` set.
- PWA: manifest registered, single SVG icon, installable, theme color set.
- Service worker: shell pre-cache, cache-first for static, network-first for
  navigations, offline fallback to `/index.html`. Manual-update flow with
  banner — no skipWaiting/clients.claim by default.
- `?nosw=1` URL flag bypasses SW registration for dev.
- Online/offline banner via `navigator.onLine`.
- Toast component, spinner/progress banner, generic confirm dialog.
- HTML escaping helper to keep extracted text safe to render.
- Emoji-cover fallback by title-keyword match (16 categories).

## Locked decisions (per Phase A "Decision points")

- IDB schema is locked at v1 with 5 stores + indexes listed above.
- ID format: `book_<timestamp_ms>` for books, `<bookId>_ch_<index>` for
  chapters. These match the legacy convention so Phase I migration to
  `sources` is mechanical.
- `pdfData` is **not** stored — Phase A keeps only extracted text. We may
  revisit in Phase M if PDF re-OCR or PDF viewer needs the original bytes.
  This keeps Phase F sync footprint small.
