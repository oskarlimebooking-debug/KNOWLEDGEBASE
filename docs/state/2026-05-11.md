### `src/data/types.ts` ‚Äî Entity interfaces

All eight IDB store shapes typed as TypeScript interfaces:

- `Book` ‚Äî id, title, author, addedAt, updatedAt, totalChapters, coverImage, kind, sizeBytes, detectionMethod. Optional: `pdfData`, `pdfRef`, sync flags.
- `Chapter` ‚Äî id, bookId, index, title, content, wordCount.
- `Progress` ‚Äî id (`{chapterId}__{date}`), chapterId, bookId, date, ts.
- `Generated` ‚Äî id, chapterId, type, data (unknown), generatedAt.
- `SettingRow` ‚Äî key, value.
- `DiscoveryResult` ‚Äî id, batchId?, title, abstract, url, authors, year, score (1‚Äì5), reason, starred, dismissed, ts, queries?, bookContext?.
- `DiscoveryCache` ‚Äî id, query, results, ts.
- `ResearchFeedback` ‚Äî id, action, resultId, ts.
- `StoreMap` ‚Äî union map from `StoreName` ‚Üí record type (enables generic typed DB operations).

### `src/data/db.ts` ‚Äî Typed IDB wrapper

Mirrors the monolith's raw IDB layer with full TypeScript types:

- `openDB()` ‚Äî singleton promise; creates all 8 stores + indexes on upgrade.
- `_resetDB()` ‚Äî resets the promise for test isolation.
- `dbPut<S>`, `dbBulkPut<S>`, `dbGet<S>`, `dbGetAll<S>`, `dbGetByIndex<S>`, `dbDelete`, `dbClear` ‚Äî all generic over `StoreName`, return `StoreMap[S]` types.
- Same schema as the monolith (DB name `ChapterWiseDB`, version `2`) ‚Äî zero migration needed.

### `src/data/books.ts` / `chapters.ts` / `progress.ts` / `generated.ts` / `settings.ts` / `discovery.ts`

Repository modules with domain-specific functions:

- `books.ts`: `getBook`, `getAllBooks` (sorted newest-first), `saveBook`, `saveBooksMany`, `deleteBook`.
- `chapters.ts`: `getChapter`, `getChaptersByBook` (sorted by index), `getAllChapters`, `saveChapter`, `saveChaptersMany`, `deleteChapter`.
- `progress.ts`: `markChapterComplete` (idempotent, creates `{chapterId}__{date}` id), `getProgressByBook`, `getAllProgress`, `getReadDates` (unique sorted dates).
- `generated.ts`: `genKey`, `getGenerated`, `getGeneratedByChapter`, `saveGenerated`, `cacheGenerated` (build + save in one call), `deleteGenerated`.
- `settings.ts`: `getSetting` (overloaded ‚Äî returns typed default for built-in keys, accepts fallback), `setSetting`, `DEFAULTS`.
- `discovery.ts`: result CRUD, batch lookup, cache entry CRUD, feedback CRUD.
