### `src/providers/ai/types.ts` — CallAIPlugin interface

```ts
interface CallAIPlugin {
  id: string;
  displayName: string;
  isConfigured(): Promise<boolean>;
  ensureAuth(): Promise<void>;
  call(prompt: string, options?: AICallOptions): Promise<string>;
  listModels?(): Promise<AIModelInfo[]>;
}
```

`AICallOptions`: `model`, `temperature`, `maxOutputTokens`, `jsonMode`, `timeoutMs`,
`images`, `onToken`, `signal`.

### `src/providers/ai/` — Five provider implementations

- **`gemini.ts`** (`GeminiProvider`): reads `geminiApiKey` + `selectedModel` from settings.
  Full `call()` with AbortController timeout, JSON mode, error unwrapping.
  `listModels()` hits the Gemini models endpoint, falls back to 5 hardcoded models.
  `anySignal()` helper for combining external + timeout signals.

- **`merlin.ts`** (`MerlinProvider`): Firebase email/password auth (`authenticate()`), token
  refresh on expiry. `call()` posts to Merlin unified thread API and consumes SSE stream,
  accumulates tokens, fires `onToken` callbacks.

- **`junia.ts`** (`JuniaProvider`): bearer-token chat. `call()` POST to Junia ai-chat, strips
  trailing UUID artifact from response.

- **`perplexity.ts`** (`PerplexityProvider`): routes through `/api/perplexity/proxy` Vercel edge
  function. Passes `model`, `temperature`, `max_tokens`.

- **`docanalyzer.ts`** (`DocAnalyzerProvider`): configured-check only; `call()` throws with
  guidance to switch providers. Placeholder until Phase R.

### `src/providers/ai/registry.ts` — Provider registry + callAI

- `aiProviders`: `Record<string, CallAIPlugin>` seeded with all five providers.
- `registerAIProvider(plugin)`: replaces or adds an entry (used in tests for mocks).
- `getActiveProviderId()`: reads `aiProvider` setting, defaults to `'gemini'`.
- `getActiveProvider()`: resolves the active plugin, throws on unknown id.
- `callAI(prompt, opts?)`: single entrypoint for all AI calls — delegates to active provider.
- `hasAnyAIProvider()`: returns `true` if the active provider reports `isConfigured()`.

### `src/providers/tts/types.ts` — CallTTSPlugin interface

```ts
interface CallTTSPlugin {
  id: string;
  displayName: string;
  isConfigured(): Promise<boolean>;
  synthesize(text: string, options?: TTSCallOptions): Promise<Blob>;
  listVoices?(): Promise<TTSVoice[]>;
}
```

`TTSVoice`: `{ id, name, language?, preview? }`.

### `src/providers/tts/` — Two TTS provider implementations

- **`lazybird.ts`** (`LazybirdProvider`): configured when both `lazybirdApiKey` and
  `lazybirdVoice` are set. `synthesize()` POSTs to `https://api.lazybird.app/v1/generate-speech`,
  returns raw `Blob`. `listVoices()` GETs the voices list.

- **`browser.ts`** (`BrowserTTSProvider`): thin wrapper for Web Speech API. `isConfigured()`
  checks for `window.speechSynthesis`. `synthesize()` intentionally throws (no Blob output).
  `listVoices()` wraps `getVoices()` + the `voiceschanged` event.

### `src/providers/tts/registry.ts` — TTS registry + callTTS

- `ttsProviders`: `{ lazybird, browser }`.
- `registerTTSProvider(plugin)`: test hook.
- `getActiveTTSProviderId()`: reads `ttsProvider` setting, defaults to `'browser'`.
- `callTTS(text, opts?)`: delegates to active TTS provider's `synthesize()`.
