### Phase E — TTS state and constants (E1 / E3 / E14)

- `ttsState` — shared playback state: `provider`, `chapterId`, `isPlaying`, `sentences[]`,
  `currentSentence`, `speed`, `audio`.
- `ppState` — persistent player meta: `isActive`, `chapterId`, `chapterTitle`, `bookTitle`,
  `provider`.
- `seqState` — sequential queue: `isActive`, `bookId`, `chapterIds[]`, `currentIndex`.
- `LAZYBIRD_API_BASE = 'https://api.lazybird.app/v1'`.

### Phase E — TTS clean prompt (E2)

- `TTS_CLEAN_PROMPT` template: strips markdown formatting (**, *, #, code fences), citations
  ([1], (Smith 2020)), URLs, OCR artifacts, converts tables to prose. ~450 chars.
- `cleanTextForTTS(chapter, force)`:
  - Cache lookup → `tts_cleaned_<chapterId>` in `generated` store.
  - If no Gemini key, falls back to `simpleStripForTTS` (regex-only, offline).
  - AI path: `callGeminiAPI(..., { temperature: 0.15, maxOutputTokens: 16384 })`.
- `simpleStripForTTS(text)`: regex-only clean (headings, bold/italic, markdown links,
  numeric citations, URLs). Used as offline fallback throughout.

### Phase E — Sentence splitter (E1)

- `splitIntoSentences(text)`: splits on sentence-ending punctuation followed by
  whitespace + capital letter, or on blank lines. Filters out fragments ≤ 2 chars.

### Phase E — Browser SpeechSynthesis player (E1)

- `playBrowserTTS(sentences, startIndex)`: sentence-by-sentence queue via
  `SpeechSynthesisUtterance`. `_synthCancel` flag for clean stop. `utter.rate`
  picks up `ttsState.speed`. `onend` chains to next sentence; `onerror` skips
  non-fatal errors. Calls `_onChapterEnded()` after last sentence.
- `stopBrowserTTS()`: sets `_synthCancel = true`, calls `speechSynthesis.cancel()`.
- `pauseBrowserTTS()`: calls `speechSynthesis.pause()`.
- `resumeBrowserTTS()`: calls `speechSynthesis.resume()` if paused, or re-speaks
  from `currentSentence` if synthesis was cancelled.

### Phase E — Lazybird provider (E3)

- `lazybirdGetVoices(apiKey)` → `GET /voices` with `X-API-Key` header.
- `lazybirdGenerateSpeech(text, voiceId, apiKey)` → `POST /generate-speech`.
- `blobToDataURL(blob)` → FileReader-based base64 data URL.
- `generateLazybirdAudio(chapter, opts)`:
  - Cache key `lazybird_audio_<chapterId>` in `generated` store.
  - Cleans text via `cleanTextForTTS` first.
  - Stores `{ audioBase64, voiceId }` + timestamp.
- `getSingleAudio()` → iOS-friendly singleton `Audio` element (lazy init).

### Phase E — Persistent mini-player (E5)

- Fixed `#persistent-player` div at `bottom: 0` with two-row layout:
  - Row 1 (`.pp-meta`): chapter title, book title, close button.
  - Row 2: `<input type="range" class="pp-progress">` progress bar.
  - Row 3 (`.pp-controls`): ⏮ Prev · ⏪ Back · ▶/⏸ Play/Pause · ⏩ Fwd · ⏭ Next ·
    speed button (1×) · time display.
- `showPersistentPlayer(...)`: sets title/subtitle, adds `.active` class,
  sets `document.body.style.paddingBottom = '140px'` so content scrolls above.
- `hidePersistentPlayer()`: stops all playback, removes `.active`, resets padding,
  clears `seqState.isActive`.
- `_updatePPProgress()`: for browser TTS shows `sentence N / total`; for lazybird
  shows `M:SS / M:SS` via `_fmtTime()`.
- `_startPPInterval()` / `_stopPPInterval()`: 500 ms `setInterval` for progress updates.
- Progress bar `input` event: seeks by sentence index (browser) or `currentTime` (lazybird).

### Phase E — MediaSession integration (E6)

- `_updateMediaSession(title, artist)`: sets `MediaMetadata`, registers handlers for
  `play`, `pause`, `seekbackward`, `seekforward`, `previoustrack`, `nexttrack`.
  Wrapped in `try/catch` for environments without full MediaSession support.
- Called from `showPersistentPlayer` and re-called on iOS `visibilitychange`.

### Phase E — iOS recovery (E7)

- `_setupIOSRecovery()` called in `boot()`:
  - `document.visibilitychange`: re-syncs `ttsState.isPlaying` from `audio.paused` when
    tab becomes visible. Re-registers MediaSession.
  - `window.pageshow` with `event.persisted`: same recovery for BFCache restores.
  - Audio element `onpause`/`onplay` keep `ttsState.isPlaying` in sync at all times.

### Phase E — Playback control helpers

- `_togglePlayPause()`: dispatches to browser pause/resume or audio.pause/play.
- `_skipBack()`: browser → `currentSentence - 3`; lazybird → `currentTime - 10s`.
- `_skipFwd()`: browser → `currentSentence + 3`; lazybird → `currentTime + 10s`.
- `_cycleSpeed()`: cycles `[0.75, 1, 1.25, 1.5, 2]`, updates player button label,
  adjusts `audio.playbackRate` or restarts browser TTS at new rate.

### Phase E — Sequential playback + auto-advance (E8 / E9)

- `_onChapterEnded()`: if `seqState.isActive`, advances queue; otherwise calls
  `_autoAdvanceSingle()`.
- `_autoAdvanceSingle()`: fetches next chapter in book order; if lazybird provider and no
  cached audio → stops with toast. Otherwise `_startChapterPlayback(nextCh.id)`.
- `_prevChapter()` / `_nextChapter()`: skip within the sequential queue.
- `_startChapterPlayback(chapterId)`: stop current, resolve chapter+book, dispatch to
  browser or lazybird path.
- `_playLazybirdCached(audioBase64, ...)`: loads audio src, wires `onended`/`onpause`/
  `onplay`, calls `showPersistentPlayer`, starts progress interval, calls `audio.play()`.

### Phase E — Listen mode renderer
