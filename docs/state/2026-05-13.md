### `src/pipeline/chapter.ts` — Chapter extraction orchestration

```ts
export type DetectionMethod = 'chapter' | 'part' | 'section' | 'numeric' | 'wordcount';

export interface ChapterDraft {
  title: string;       // capped at 80 chars
  content: string;
  wordCount: number;
}

export interface ExtractChaptersResult {
  chapters: ChapterDraft[];
  detectionMethod: DetectionMethod;
}

export function extractChaptersFromText(
  text: string,
  opts?: { wordsPerChapter?: number },
): ExtractChaptersResult
```

Strategy: tries `detectChapterPatterns` (from `src/lib/`) first. If it yields ≥ 2 valid
chapter slices, returns those with the pattern name as `detectionMethod`. Otherwise
falls back to `splitIntoChapters` + `deriveTitle` with `detectionMethod: 'wordcount'`.

This is the pure, tested version of the orchestration that lives in `addBookFromFile`
in the monolith (lines ~2471–2488). No IDB, no browser APIs, no side effects.

### `src/pipeline/clean.ts` — TTS text cleaning (sync)

```ts
export function simpleStripForTTS(text: string): string
```

Pure sync Markdown stripper. Removes ATX headings, bold/italic markers, inline code,
Markdown links (keeps link text), citation brackets (`[1]`, `[2, 3]`), bare URLs, and
collapses 3+ blank lines to 2. Trims result. No network calls.

This is the typed, testable extraction of `simpleStripForTTS` from the monolith (line
~4983). The async AI-powered `cleanTextForTTS` (requires IDB + AI provider) remains
in the monolith until the data layer and provider layer are fully wired into a new UI.

### `src/pipeline/feed.ts` — Feed JSON facade

```ts
export function extractFeedPosts(raw: string): unknown[]
export function looksLikeFeedJson(raw: string): boolean
```

`extractFeedPosts`: delegates to `parseFeedJsonSync` (already in `src/lib/`) and unwraps
the `posts` array. Returns `[]` on any failure. Covers sync Stages 1+2 (markdown fence
strip + bracket-depth scan). Async Stage 3 (AI repair) stays in the monolith.

`looksLikeFeedJson`: cheap heuristic (`includes('"posts"')` + opening brace or fence)
that lets callers decide whether to bother with an AI repair request.
