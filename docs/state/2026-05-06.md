### Provider layer (B1, B12)

- `callGeminiAPI(prompt, apiKey, modelOverride, options)` —
  POST to `/v1beta/models/<model>:generateContent`. 120 s
  `AbortController` timeout. Honors `options.jsonMode`,
  `options.temperature`, `options.maxOutputTokens`. Throws with the API's
  error message on non-2xx, `(finishReason: …)` on empty candidate, and
  a friendly "request timed out" message on abort.
- `fetchAvailableModels(apiKey)` — lists `models?` filtered to those
  starting with `gemini`. Falls back to a hard-coded list of five model
  names on any failure.
- `getSelectedModel()` reads `selectedModel` setting (default
  `gemini-2.5-flash`). `getGeminiKey()` reads `geminiApiKey`.
- `hasAnyAIProvider()` gate (true when a Gemini key is present). Used by
  every AI mode to render a "needs key" empty state instead of crashing.
- All errors surface as a toast + an in-mode error block with a Retry
  button (B12 acceptance).

### Prompts as data (B2)

- `DEFAULT_PROMPTS` dict with five entries: `summary`, `quiz`,
  `flashcards`, `teachback`, `formatText`. (`chapterSplit` from spec was
  not added — pattern detection is non-AI and lives outside this dict;
  may revisit if AI-enhanced titles get implemented.)
- `getPrompt(key)` returns `prompt_<key>` setting if non-empty, else the
  default. `fillTemplate(tmpl, vars)` does `{{title}}`, `{{content}}`,
  `{{userText}}` substitution.
- Settings modal exposes a textarea per prompt with a "Reset all to
  default" button (clears the override fields; saves on close).

### parseSafeJson + JSON cleanup

- `parseSafeJson(raw)` handles three failure modes Gemini exhibits:
  1. ```` ```json … ``` ```` fences,
  2. trailing prose ("Sure! { … } hope this helps."),
  3. trailing commas / unbalanced braces — walks character-by-character
     tracking string state and brace depth.
- Verified live in the running app: fenced, trailing-prose, and direct
  inputs all parse to the right object.

### Cache-first generation pattern (B4)

- `loadGenerated(type, chapter, generator, opts)` — checks the
  `generated` store for `<type>_<chapterId>`; on miss runs the generator
  and stores the row. `opts.force` re-runs and overwrites.
- `clearGenerated(type, chapterId)` — used by Regenerate buttons.
- IDs match the legacy convention so Drive sync (Phase F) and Phase I
  source migration are mechanical.

### Modes (B5–B8)

- **Summary** — JSON `{ keyConcepts[], summary, difficulty (1-5),
  readingTime (min) }`. Renders meta row (reading time + difficulty
  stars), key-concept pills, markdown-rendered summary. Difficulty is
  written back onto the chapter row so a future library card can show
  stars.
- **Quiz** — generates 5 questions (3 MC, 1 T/F, 1 open-ended). One
  question at a time; selecting an option immediately reveals correct
  answer + explanation. End-of-quiz score card; "Retake wrong only"
  button rebuilds with the wrong subset; "+ Generate more" appends 5
  fresh questions with an avoid-list. Score history persists in
  `generated` as `quiz_scores_<chapterId>` per spec.
- **Flashcards** — 6–8 cards, 3D flip animation (CSS `transform:
  rotateY`), prev/next/flip controls, "+ More cards" appends with
  avoid-list dedupe.
- **Teach-Back** — textarea + "Submit for grading" → score + strengths /
  gaps / suggestions. Cached with the user's text so revisits show their
  last attempt.
- Every mode uses the same shape: `renderAIMode(host, ctx, type, label,
  generator, viewFn)` which handles the needs-key check, loading state,
  error state with retry, and a Regenerate bar at the bottom.

### Markdown renderer + sanitizer (B9)

- `mdRender(src)` — in-house renderer covering headings (h1–h6), bold,
  italic, code (inline + fenced), bullet/ordered lists, blockquotes,
  links (auto-`target=_blank rel=noopener`).
- `sanitizeHtml(html)` — whitelist sanitizer. Walks the parsed DOM:
  any tag not in the allow list (`h1-6, p, br, strong, em, b, i, u, ul,
  ol, li, blockquote, pre, code, a`) becomes its text content; only
  `href/target/rel` survive on `<a>`; non-`http(s)` hrefs collapse to
  `#`. Verified live: `<script>alert(1)</script>` becomes inert text;
  `<img src=x onerror=alert(1)>` is stripped completely.
