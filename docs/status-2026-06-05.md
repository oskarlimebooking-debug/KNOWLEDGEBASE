## Feature details

### Edit flow
1. User clicks the note text in a freeform chip in the Book Notes panel
2. Text div is replaced by a focused `<textarea>` pre-filled with the full note
3. **Enter** (not Shift+Enter): trims, validates non-empty, saves via `dbPut('annotations', ann)`, shows "Note updated ✓" toast, re-renders panel
4. **blur**: same as Enter (auto-save on focus-out)
5. **Escape**: cancels without IDB write, re-renders panel restoring original text
6. Empty text → "Note cannot be empty" error toast, focus returned to textarea

### UX details
- `title="Click to edit"` tooltip on hover signals editability
- `ann-chip-freeform-text` class shares the cursor/hover CSS already used in the sidebar (`.ann-chip-freeform-text { cursor: text; }`) — no new CSS needed
- Clicking the chip outside the text area still navigates to the chapter as before
- If note text unchanged (no edit), no IDB write is made, no toast shown
