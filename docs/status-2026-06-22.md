# Headway — Status 2026-06-22

## Current phase: N50 ✅ COMPLETED TODAY

### What was built today

**Phase N50 — Live progress refresh after mark/unmark** (~18 lines added):

Fixed the stale `#chapter-progress` display (the "Ch X of Y · Z% done" span in the chapter header) after toggling chapter completion while staying in the chapter view.

**Root cause**: `openChapter` populates `#chapter-progress` from a fresh IDB read, but neither the button handler nor the M-key handler refreshed it after mark/unmark — only the button text changed.

**Fix — three changes**:

1. **New helper** `_refreshChapterProgressDisplay()` (added after `bookCompletion`): reads fresh progress from IDB for `App._chapterCtx.bookId`, recomputes the done-%, and writes the updated "Ch N of T · P% done · ~M min" text to `#chapter-progress` in-place.

2. **Button handler** (line ~7153): added `await _refreshChapterProgressDisplay()` after `unmarkChapterComplete` (mark always navigates away, so only unmark needs the refresh there).

3. **M-key handler** (line ~8480/8485): added `await _refreshChapterProgressDisplay()` after BOTH mark AND unmark, because the keyboard shortcut never navigates — it just toggles the button text in-place.
