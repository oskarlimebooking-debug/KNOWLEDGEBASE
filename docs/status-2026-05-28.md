## Feature details

### Interaction
- **Double-click** a tag chip in the Deck Manager filter bar
- Chip text is replaced by a transparent inline input pre-filled with the current tag name
- **Enter** or **click away (blur)**: commits the rename
- **Escape**: cancels, reverts to normal chip rendering
- Single-click still filters (220 ms debounce, imperceptible in practice)

### Rename logic
- Tag names are lowercased and capped at 30 characters (same as tag creation)
- If the new name equals the old name (or is empty): no-op, just re-renders
- If the new name already exists in `allTags`: shows error toast, re-renders
- If valid: calls `_renameTag(oldTag, newTag)` → bulk-updates IDB → shows toast `Renamed "x" → "y" on N cards` → re-renders deck manager
- `_deckTagFilter` is updated to the new name if it matched the old name (filter persists)

### Key bug fix
The initial implementation fired `ondblclick` on an already-detached node because `onclick` synchronously called `renderDeckMgr` (rebuilding the DOM). Fixed by debouncing `onclick` with a 220 ms `setTimeout`; `ondblclick` calls `clearTimeout` to cancel the pending re-render before it fires.
