## Feature details

### Collection pill bar

Appears **above the book grid** when at least one collection exists. Always starts with **"All (N)"** pill selected. Each named collection gets a coloured dot + book count.

```
[All (12)]  [● Work (3)]  [● Fiction (5)]  [⊞ Manage]
```

Clicking a pill filters the library in-place (no page reload). Clicking "All" resets the filter.

### Collection manager modal

Opened via "⊞ Manage" button in the pill bar. Shows:
- List of existing collections (dot, name, book count, × delete button)
- Colour picker row (6 swatches; selected swatch has a text-coloured border)
- Name input + "Add" button (Enter also submits)

Deleting a collection removes it from IDB and clears `collectionId` on all affected books (books are not deleted). Confirmation dialog shown before delete.

### Book assign context menu

**Right-click any book card** → floating popup appears at cursor with:
- "No collection" (unassign)
- Each existing collection with its dot and name
- The current assignment is bold/accented

Clicking assigns immediately, re-renders the library, and closes the popup. Clicking outside also closes it.

### IDB schema

`collections` store: `{ id: uuid, name: string, color: string, createdAt: ISO-string }`  
`books` store: existing; gains `collectionId: string | null` field (optional, no migration needed — undefined treated as null).
