## Feature details

### N7 — Word definitions

**Two trigger pathways:**

1. **Double-click** (desktop): native `dblclick` event on `#mode-host`; browser auto-selects the word; if it's a single token (no whitespace, ≤ 40 chars), `defineWord` is called immediately at mouse position.

2. **Annotation menu Define button** (mouse + touch): when user drags to select a single word and the annotation context menu opens, a "📖 Define" button appears alongside "+ Note". Clicking it dismisses the menu and opens the definition popup.

**Popup structure:**
```
┌─────────────────────────┐
│ granular              × │
│ ADJECTIVE               │
│ Consisting of, or       │
│ resembling granules...  │
│   a granular substance  │
│ Grainy.                 │
│   It has a granular...  │
└─────────────────────────┘
```

**Wiktionary REST API:** `GET https://en.wiktionary.org/api/rest_v1/page/definition/{word}`
- Word lowercased, non-alpha stripped before lookup
- Response parsed: `json['en']` → up to 2 parts-of-speech → up to 2 definitions each
- Definition HTML tags stripped client-side
- Falls back to first available language if no `en` key

**Popup positioning:** fixed, clamps to viewport (left ≥ 8px, right ≤ vw−316px, top ≥ 8px + scrollY).

**Dismiss:**
- × button inside popup
- Click anywhere outside popup
- Leaving the chapter (unbindAnnSelection calls _hideDefinePopup)

## Known gaps / follow-ups

1. **No definition for acronyms/proper nouns** — Wiktionary lacks many technical terms and names; a graceful "No definition found" message is already rendered.
2. **Language selector** — Wiktionary can return multi-language results; currently shows `en` or first available. A language toggle could be added later.
3. **Mobile long-press** — The double-click trigger doesn't fire on mobile (touch devices use tap/long-press). The annotation menu "Define" button works on touch. A dedicated long-press detector (`touchstart` + 500ms timer) could be added for mobile.
