## Feature details

### Select mode flow
1. Header shows `☑ Select` button (secondary style)
2. User clicks → button becomes `✕ Cancel` (accent background); select bar appears at top of card list; each card gets a left-side checkbox; cards get `padding-left: 38px`
3. **Checkbox click** → card border turns accent-colored; count label updates ("N selected"); "Delete (N)" button enables
4. **Card body click** → also toggles checkbox (entire card is clickable in select mode)
5. **Select all checkbox** → selects/deselects all visible cards (respects search + tag filter); indeterminate state shown when partially selected
6. **Delete (N)** → bulk `dbDelete` all selected IDs → "Deleted N cards" toast → exits select mode → re-renders with updated count
7. **✕ Cancel** or **Escape** → exits select mode, re-renders without deleting, deck manager stays open

### Guard behavior
- Edit and individual Delete buttons still work in select mode (click guard via `el.onclick` only fires when `e.target !== cb`)
- Select state is scoped to currently visible cards only (filter + tag filter aware via `nSelected` counter on render)
- `_deckSelected` persists across search filter changes (you can select in "vocab" filter, switch filter, and previously-selected IDs remain in the Set)
