## Feature details

### Rename flow
1. User **double-clicks** a named collection pill (e.g., "Science (0)")
2. Pill content is replaced: color dot + inline input pre-filled with "Science", auto-focused and selected
3. User edits the name
4. **Enter** (or clicks away/blur): validates non-empty, checks for duplicate names (case-insensitive), calls `saveCollection({ ...coll, name: newName })`, shows toast `Renamed "Science" → "Natural Sciences"`, re-renders bar
5. **Escape**: cancels edit, re-renders bar with original name, no IDB change
6. Single-click still filters the library (220ms debounce means rapid dblclick cancels the filter action)

### Guard behavior
- "All" pill has no `data-coll-id` so gets no dblclick handler — only named collections are renameable
- Duplicate name guard: if new name matches an existing collection (case-insensitive), shows error toast and re-renders without saving
- Empty name guard: blank input is treated as cancel (no IDB write, just re-render)
- 40-char max enforced via `.slice(0, 40)` on commit
