## Feature details

### Phase N19 — Inline note cards

**Two entry points:**

#### 1. Annotation sidebar (`🖊` button)
- Every chip shows a `＋ Card` button in the bottom-right of the chip footer
- **Annotation WITH note**: one click → card created (`front = text`, `back = note`), button becomes `✓ In deck`
- **Annotation WITHOUT note**: click reveals an inline textarea (`Back of card…`); user fills in the back, clicks "Save card"; card created, form collapses, button enrolled
- On re-open, enrolled state is re-checked from IDB so the deck badge is always accurate
- Click on the chip body (outside button/form) still navigates to the highlight in the text and closes sidebar

#### 2. Annotation popup (clicking a `<mark>`)
- Popup now has three action buttons: `＋ Card`, `Delete`, `Close`
- Same with/without-note logic: with note → one click; without note → textarea appears inline in the popup
- Enrolled check runs asynchronously when the popup opens (matches sidebar behaviour)

**Card data shape:** same as AI-generated cards — `srsAddCard(chapterId, bookId, front, back)` — so they appear in the review deck, due badges, and Deck Manager with no special handling.
