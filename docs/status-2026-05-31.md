## Feature details

### Inline edit flow
1. User sees note chip: `✏️ NOTE` label + note text (cursor:text, hover tint hints it's editable)
2. User clicks the note text body → text div replaced by a focused textarea pre-filled with note content, all text selected
3. **Enter (no Shift)** → trim + validate → if non-empty and changed, `dbPut` saves; re-render sidebar showing updated text; "Note updated ✓" toast
4. **Enter on unchanged text** → re-render silently (no write, no toast)
5. **Escape** → cancel edit, re-render with original text
6. **Blur (tab/click away)** → same as Enter (save)
7. **Empty and save** → `showToast('Note cannot be empty', 'error')`; focus stays on textarea
8. Clicking the `✏️ NOTE` label or chip footer → scrolls to position + closes sidebar (unchanged)
9. Clicking Delete button → still works (guard in click handler prevents conflict)

### Guard logic
```
chip click:
  if target is .ann-chip-card-btn / .ann-card-quick-form / .ann-freeform-del / .ann-chip-edit-ta → return (no-op)
  if freeform chip:
    if target is .ann-chip-freeform-text → _startFreeformEdit(chip, ann)
    else → scroll + close sidebar
  else → existing highlight-chip scroll behaviour
```
